    内存的不同用途

    首先科普一个概念，什么是缓冲区，什么是缓冲区溢出？

    缓冲区：缓冲区(buffer)，它是内存空间的一部分。也就是说，在内存空间中预留了一定的存储空间，这些存储空间用来缓冲输入或输出的数据，这部分预留的空间就叫做缓冲区，显然缓冲区是具有一定大小的。

    缓冲区溢出：再大缓冲区中的数据向小缓冲区复制的过程中，没有注意小缓冲区的边界，“撑爆”了小的缓冲区，从而冲掉了和小缓冲区相邻内存区域的其他数据而引起的内存问题。

    在不同的操作系统中，一个进程可能会被分配到不同的内存区域中，无论是什么操作系统，什么样的架构，大致可以把内存分为四区：

    • 代码区：存放二进制机器代码，处理器在这个区域取指令并执行
    • 数据区：全局变量等
    • 堆区：进程可以在堆区申请一定大小的内存，并在使用完毕后重新还给堆区，东区可实现动态的内存回收和分配
    • 栈区：用于动态的存储函数之间的调用关系，以保证被调用函数在返回时恢复到母函数中继续执行

    在函数调用中，伴随的系统栈中的操作如下：

    • 在main函数中调用func_A的时候，手下你在自己的栈帧中压入函数返回的地址，然后为func_A创建新栈帧并压入系统栈

    • 在func_A调用func_B的时候，永阳现在自己的栈帧中压入函数的返回地址，然后为func_B创建新栈帧并压入系统栈
    • 在func_B返回时，func_B的栈帧被弹出系统栈，func_A栈帧中的返回地址被“裸漏”在站定，此时处理器按照这个返回地址重新跳到func_A代码区中执行
    • 在func_A返回时，func_A的栈帧被弹出系统栈，main函数战阵中的返回地址“裸漏”在站定，此时处理器按照这个返回地址跳到main函数代码区中执行

    寄存器与函数栈帧

    每一个函数独占自己的栈帧空间。当前正在运行的函数的栈帧总是在站定。Win32系统提供了两个特殊的寄存器用于标识位于系统栈顶端的栈帧。

    • ESP：栈指针寄存器，其内存放一个指针，该指针永远指向系统栈最上面一个栈帧的栈定。
    • EBP：基址指针寄存器，其内存放一个指针，该指针永远指向系统栈最上面一个栈帧的底部。
    • EIP：指令寄存器，其内存放一个指针，该执行永远指向下一条等到执行的指令地址。

    EBP 指向当前位于系统栈最上边一个栈帧的底部，而不是系统栈的底部

    函数栈帧：ESP 和 EBP 之间的内存空间为当前栈帧，EBP 标识了当前栈帧的底部，ESP标识了当前栈帧的顶部。

    在函数栈帧中，一般包含几类重要信息：

    • 局部变量：为函数局部变量开辟的内存空间。
    • 栈帧状态值：保存前栈帧的顶部和底部(实际只保存前栈帧的底部，前栈帧的顶部可以通过堆栈平台计算得到)，用于在本栈帧被弹出后恢复出上一个栈帧。
    • 函数返回地址：保存当前函数调用前的“断点”信息，也就是函数调用前的指令位置，以便在函数返回时能够恢复到函数被调用前的代码区中继续执行执行。

    函数栈帧大小不固定，一般与其对应函数的局部变量多少有关

    • 参数入栈：将参数从右往左依此压入系统栈中
    • 返回地址入栈：将当前代码区调用指令的下一条指令地址压入栈中，共函数返回时继续执行
    • 代码区跳转：处理器从当前代码区跳转到被调用函数的入口点
    • 栈帧调整：
      • 保存当前栈帧状态值，以备后面恢复本栈帧时使用（EBP入栈）
      • 将当前栈帧切换到新栈帧（将ESP值装入 EBP，更新栈帧底部）
      • 给新栈帧分配空间（把ESP减去所需要的空间的大小，抬高栈顶）

    更强大、更通用的攻击通过缓冲区溢出改写的目标往往不是某一个变量，而是修改栈帧最下方的EBP和返回返回地址等栈帧状态值。
