段的定义

段的概念

Win32的Hello源程序中的语句归纳：

编译选项、模式等定义并不会在编译好的可执行程序中产生什么，它们只是“说明”，而真正的数据和代码是定义在各个段中的

.stack、.data、.data?、.const和.code是分段微指令，Win32中实际上只有代码和数据之分，.data、.data?、.const都是数据段，.code是代码段，Win32汇编不必考虑堆栈。

Win32中的段指的是内存的“分段”。上一个段的结束就是下一个段的开始，所有的“分段”结合起来，包括系统使用的地址空间，就组成了整个可以寻址的4GB空间。Win32环境的内存使用了分页机制，每个页4KB并且可以自由指定属性，所有上一个4KB可能是代码，属性是可执行但不可写，下一个4KB就有可能是即可读可写但不可知性的数据。

数据段

.data、.data?、.const定义的是数据区，分别对应不同方式的数据定义，最后生成的可执行文件也分别放在不同的节区（section）中。程序中的数据定义一般被归纳分三类：

  • 第一类是可读可写的已定义变量。这些数据在源程序中已经被定义了初始值，而且在程序的执行中可能被改变。
  • 第二类是可读可写未定义变量。这些变量一般都是当作缓冲区或者程序执行后才开始使用的，这些数据可以定义在.data段中，也可以定义在.data？段中，但是一般把它放到.data?段中。如果要定义一个100KB的缓冲区，可以用以下的语句定义：

如果这个语句放在.data段中，编译器就会认为这些数据在程序装入的时候就必须有效，所以生成的执行文件部分就保留了所有100KB的内容，即使全部为0。

如果这个语句放在.data?段中，其中的内容会被编译器认为在程序执行后才用到，所以在可执行文件生成的时候只保留了大小信息，不会为其浪费磁盘空间。

.data?段是未初始化数据段，其中的数据也是可读可写的，但在可执行文件中不占用空间，.data?段在可执行文件中一般存放在_BSS节区中

  • 第三类数据是一些常量。比如一些需要显示的自接触信息，它们在装入程序的时候已经有效，但在整个执行过程中不需要修改，这些数据可以放在.const段中，.const段是常量段，它是可读不可写的。可以把常量放入任意段，但是前提是这个段是可读的。

代码段

.code段是代码段，所有的指令必须写在代码段中，在可执行文件中，代码段一般是存放在_TEXT节区中的。Win32环境中的数据段是不可知性的，只有代码段有可执行的属性。

堆栈段

在程序中不必定义堆栈段，系统是会自动分配堆栈空间。堆栈段的内存属性是可读写并且可执行的，这样靠动态修改代码的反跟踪模块可以拷贝到堆栈中的去便修改边执行。

程序结束和程序入口

在c语言中，程序不需要指定程序由哪个函数开始执行，编译器已经约定好了开始执行的函数，而在汇编程序中，并没有main函数，程序员可以指定从任意一个代码段的任意一个地方开始执行，最后使用end

当把多个模块链接在一起的时候，只能有一个主模块指定入口地址，在多个模块中指定入口地址或者没有一个模块指定了入口地址，链接程序都会报错。

注释和换行

汇编程序中的注释都是以;开始。

当程序过长的时候，使用反斜杠\来做换行符

调用API

API是什么

Win32程序是构筑在Win32 API基础上的。在Win32 API中，包括了大量的函数、结构和消息等，它不仅位应用系统所调用，也是Windows自身的一部分，Windows自身的运行也调用这些API函数。

Win32的系统功能模块是放在Windows的动态链接库（DLL）中，DLL是Windows的一种可执行文件，采用的是和EXE文件同样的PE格式

实际上，Win32的基础就是由DLL组成的。Win32 API的核心由3个DLL提供，它们是：

  • KERNEL32.dll：系统服务共呢个。包括内存管理、任务管理和动态链接等
  • GDI32.dll：图形设备接口。利用VGA等显示设备驱动程序完成显示问呗和矩形等功能
  • USER32.dll：用户接口服务，建立窗口和传送消息等

标号、变量和数据结构

变量的定义必须在可写的段内，如.data和.data?，或者在堆栈内。按照定义位置的不同，MASM中的变量也分为全局变量和局部变量两种。

在MASM中的标号和变量的命名规范是一致的，它们是：

  • 可以用字母、数字、下划线及符号@、$和？.
  • 第一个符号不可以是数字
  • 长度不能超过240个字符
  • 不能使用指令名等关键字
  • 在作用域内必须是唯一的

标号

标号的定义

当在程序中使用一条跳转指令的时候，可以用标号来表示跳转的目的地，编译器在编译的时候会把它替换成地址，标号可以定义在目的指令同一行的头部，也可以在目的指令前一行单独用一行定义

单个子进程中标号不能同名，使用一个标号的作用域是当前子程序，需要从一个子程序跳转到另外一个子程序的时候，使用双标号

MASM中的@@

在DOS的年代，为标号起名是很麻烦的事情，因为汇编指令用到跳转的指令特别多，任何比较和测试等都需要涉及跳转，所以在程序中有很多标号，在整个程序检测重名是一个相当麻烦的时候，所以经常出现一些奇怪的标号。

全局变量的初始化

全局变量在定义中既可以指定初始值，也可以用问号预留空间，在.data?段中，只能用问好预留空间，因为.data?不能指定初始化，预留空间在时间运行中的只是0，在程序中可以不必特地为它赋值。

局部变量

局部变量的作用域是单个子程序，在进入子程序的时候，通过修改堆栈指针esp来预留出需要的空间，在用ret指令返回主程序前，通过通过回复esp丢弃这些空间，这些空间就随之无效了。
